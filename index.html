Inicializar Proyecto (Desde CMD con XAMPP)

Sigue estos pasos en tu terminal para crear el proyecto en la carpeta de servidor local:

  1. 1. Abrir CMD: Presiona Win + R, escribe cmd y presiona Enter.
  2. 2. Salir a la raíz del disco C: 
    cd ..
            cd ..
  3. 3. Entrar a la carpeta de XAMPP: 
    cd xampp/htdocs
  4. 4. Crear el proyecto Laravel 12:

    Copia y pega el comando (cambia "mi-proyecto" por el nombre que quieras):

    composer create-project laravel/laravel:^12.0 mi-proyecto
  5. 5. Acceder a la carpeta creada: 
    cd mi-proyecto
  6. 6. Abrir en Visual Studio Code:

    Estando dentro de la carpeta, escribe el siguiente comando para abrir el editor automáticamente:

    code .
Nota: Una vez creado, recuerda que para ver tu proyecto en el navegador debes ejecutar php artisan serve o configurar un VirtualHost en XAMPP.

Configuración de Base de Datos

1. Activar Servidores en XAMPP

Abre el XAMPP Control Panel y asegúrate de que los módulos estén en verde:

  • Apache: Servidor web.
  • MySQL: Gestor de base de datos.
2. Crear Base de Datos en phpMyAdmin
  1. En tu navegador, ve a: http://localhost/phpmyadmin
  2. Haz clic en "Nueva" (New).
  3. Escribe el nombre de tu base de datos (ejemplo: db_proyecto).
  4. Haz clic en "Crear".
3. Configurar el archivo .env

Abre tu proyecto en VS Code, busca el archivo .env en la raíz y modifica estas líneas:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=db_proyecto  <-- Pon aquí el nombre que creaste
DB_USERNAME=root         <-- Usuario por defecto en XAMPP
DB_PASSWORD=             <-- Déjalo vacío (en XAMPP no tiene clave)
Importante: Después de cambiar el archivo .env, es recomendable limpiar la caché con:
php artisan config:clear
php artisan config:cache

Verificar Conexión y Crear Tablas

Una vez configurado el archivo .env, debemos comprobar si Laravel puede comunicarse con MySQL ejecutando las migraciones iniciales.

Ejecutar Migración

Abre la terminal (dentro de VS Code o en el CMD) y escribe:

php artisan migrate
¿Cómo saber si funcionó?

En la Terminal:

Preparing database...
Creating migration table...
Running migrations... DONE

En phpMyAdmin:

Refresca tu base de datos; deberías ver tablas como users, jobs, migrations, etc.

Errores Comunes:
  • "Unknown database": El nombre en DB_DATABASE no coincide con el de phpMyAdmin.
  • "Connection refused": MySQL en el Panel de XAMPP no está encendido (botón Start).
  • "Access denied": Revisa que el DB_USERNAME sea root y el password esté vacío.

Ejemplo Completo: Producto y Proveedor

1. Crear Modelos y Migraciones 
php artisan make:model Proveedor -m
php artisan make:model Producto -m
2. Migración: proveedores 
Schema::create('proveedores', function (Blueprint $table) {
    $table->id();
    $table->string('nombre');
    $table->string('correo')->nullable();
    $table->timestamps();
});
3. Migración: productos 
Schema::create('productos', function (Blueprint $table) {
    $table->id();

    $table->string('nombre');
    $table->decimal('precio', 10, 2);

    $table->unsignedBigInteger('proveedor_id');

    $table->foreign('proveedor_id')
          ->references('id')
          ->on('proveedores')
          ->onDelete('cascade');

    $table->timestamps();
});
4. Modelo Proveedor 
class Proveedor extends Model
{
    protected $fillable = ['nombre', 'correo'];

    public function productos()
    {
        return $this->hasMany(Producto::class);
    }
}
5. Modelo Producto 
class Producto extends Model
{
    protected $fillable = ['nombre', 'precio', 'proveedor_id'];

    public function proveedor()
    {
        return $this->belongsTo(Proveedor::class);
    }
}
6. Ejecutar Migraciones 
php artisan migrate
7. Mostrar Proveedores en un Select 
// Controlador
public function crear()
{
    $proveedores = Proveedor::all();
    return view('producto.crear', compact('proveedores'));
}

<select name="proveedor_id" class="form-control">
    <option value="">Seleccione un proveedor</option>

    @foreach($proveedores as $proveedor)
        <option value="{{ $proveedor->id }}">
            {{ $proveedor->nombre }}
        </option>
    @endforeach
</select>
8. Acceder al Proveedor desde Producto

Gracias a la relación belongsTo, podemos acceder al proveedor directamente desde el producto: 

// En Blade (vista)
@foreach($productos as $producto)
    {{ $producto->nombre }} - 
    {{ $producto->proveedor->nombre ?? 'Sin proveedor' }}
@endforeach

También en tabla: 

<table class="table">
    <tr>
        <th>Producto</th>
        <th>Proveedor</th>
    </tr>

    @foreach($productos as $producto)
    <tr>
        <td>{{ $producto->nombre }}</td>
        <td>{{ $producto->proveedor->nombre ?? 'Sin proveedor' }}</td>
    </tr>
    @endforeach
</table>
Relación uno a muchos: un proveedor puede tener muchos productos, pero cada producto pertenece a un solo proveedor.
Errores Comunes:
  • No definir la relación belongsTo.
  • No usar proveedor_id.
  • No usar with('proveedor') en el controlador.

Comandos Básicos de Artisan

Inicia el servidor de desarrollo local.
Normalmente corre en http://127.0.0.1:8000.

Permite crear archivos base del proyecto.
  • make:model
  • make:controller
  • make:migration

Crea automáticamente:
  • Modelo
  • Controlador
  • Migración
Ahorra tiempo y mantiene el orden del proyecto.

Ejecuta las migraciones pendientes y crea las tablas en la base de datos.

⚠️ Elimina todas las tablas y las vuelve a crear.
  • Usar solo en desarrollo
  • Cuando hay errores de migración
  • No usar en producción

Limpia todas las cachés del proyecto.
Usar cuando los cambios no se reflejan.

Limpia la caché de configuración.
Obligatorio después de modificar el archivo .env.

Guarda la configuración en caché para mejorar el rendimiento.
Recomendado en producción, no durante desarrollo activo.

Crear Módulo Producto en Laravel

En este apartado vamos a crear el módulo Producto en Laravel utilizando modelo, controlador, migración, rutas y AdminLTE paso a paso.

Crear Modelo, Controlador y Migración

En la terminal ejecutamos el siguiente comando:

php artisan make:model Producto -mc

Este comando crea automáticamente tres archivos:

  • Modelo Producto
  • Controlador ProductoController
  • Migración de la tabla productos
Ubicación de los Archivos
Modelo 
app/Models/Producto.php
Controlador 
app/Http/Controllers/ProductoController.php
Migración 
database/migrations/xxxx_create_productos_table.php
Configurar el Modelo

Ejemplo del modelo Producto.php solo con el campo nombre:


namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Producto extends Model
{
    protected $fillable = ['nombre'];
}
Configurar la Migración

En la migración agregamos el campo nombre:


Schema::create('productos', function (Blueprint $table) {
    $table->id();
    $table->string('nombre');
    $table->timestamps();
});

Luego ejecutamos:

php artisan migrate
Configurar el Controlador

En ProductoController.php solo creamos la función index:


public function index()
{
    return view('admin.producto.index');
}

Debemos crear las carpetas dentro de: 

resources/views/admin/producto/index.blade.php
Plantilla Básica en la Vista

En el archivo index.blade.php debemos colocar la plantilla básica de AdminLTE la podemos copiar y pegar en la archivo index.blade.php . 


@extends('adminlte::page')

@section('title', 'Producto')

@section('content_header')
    
Lista de Productos

@stop

@section('content')
    
Aquí va el contenido del módulo producto

@stop
Configurar Ruta

En el archivo routes/web.php agregamos:


use App\Http\Controllers\ProductoController;

Route::resource('/producto', ProductoController::class);
Verificar Rutas

Ejecutamos:

php artisan route:list

Aquí aparecerán todas las rutas del módulo producto.

Configurar AdminLTE

Ir al archivo:

config/adminlte.php

Buscar la sección menu y agregar:


[
    'text' => 'Producto',
    'route' => 'producto.index',
    'icon' => 'fas fa-box'
],
Limpiar Configuración

Ejecutar:


php artisan config:clear
php artisan config:cache
Ejecutar la Aplicación 
php artisan serve
Resultado Final
  • Modelo Producto creado
  • Migración ejecutada
  • Controlador funcionando
  • Vista con plantilla AdminLTE
  • Ruta configurada
  • Menú Producto visible en AdminLTE
  • Módulo funcionando correctamente

4. CreateProducto.php COMPLETO
<?php

namespace App\Livewire\Producto;

use Livewire\Component;
use App\Models\Producto;
use Livewire\Attributes\On;

class CreateProducto extends Component
{
  public $nombre;
    public $producto_id;
    public $edit = false;

     protected function rules()
    {
        return [
            'nombre' => 'required'
        ];
    }

     protected $messages = [
        'nombre.required' => 'El nombre es obligatorio'
    ];
    
    #[On('edit-producto')]
    public function edit($data)
    {
        $this->producto_id = $data['id'];
        $this->nombre = $data['nombre'];
        $this->edit = true;
    }

    public function save()
    {
        $this->validate();

        if ($this->edit) {
            Producto::find($this->producto_id)->update([
                'nombre' => $this->nombre
            ]);
        } else {
            Producto::create([
                'nombre' => $this->nombre
            ]);
        }

        $this->reset(['nombre','producto_id','edit']);

        $this->dispatch('producto-list-refresh');
    
    }

    public function render()
    {
        return view('livewire.producto.create-producto');
    }
}

<?php

namespace App\Livewire\Producto;

use Livewire\Component;
use App\Models\Producto;
use Livewire\Attributes\On;

class ListProducto extends Component
{
     public function edit($producto)
    {

        $this->dispatch('edit-producto', data: $producto);
    }

    #[On('producto-list-refresh')]
    public function refresh()
    {
        // Livewire refresca automáticamente
    }

    public function render()
    {
        $productos = Producto::orderBy('id','desc')->get();

        return view('livewire.producto.list-producto', compact('productos'));
    }
}
