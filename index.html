Inicializar Proyecto (Desde CMD con XAMPP)

Sigue estos pasos en tu terminal para crear el proyecto en la carpeta de servidor local:

  1. 1. Abrir CMD: Presiona Win + R, escribe cmd y presiona Enter.
  2. 2. Salir a la raíz del disco C: 
    cd ..
            cd ..
  3. 3. Entrar a la carpeta de XAMPP: 
    cd xampp/htdocs
  4. 4. Crear el proyecto Laravel 12:

    Copia y pega el comando (cambia "mi-proyecto" por el nombre que quieras):

    composer create-project laravel/laravel:^12.0 mi-proyecto
  5. 5. Acceder a la carpeta creada: 
    cd mi-proyecto
  6. 6. Abrir en Visual Studio Code:

    Estando dentro de la carpeta, escribe el siguiente comando para abrir el editor automáticamente:

    code .
Nota: Una vez creado, recuerda que para ver tu proyecto en el navegador debes ejecutar php artisan serve o configurar un VirtualHost en XAMPP.

Configuración de Base de Datos

1. Activar Servidores en XAMPP

Abre el XAMPP Control Panel y asegúrate de que los módulos estén en verde:

  • Apache: Servidor web.
  • MySQL: Gestor de base de datos.
2. Crear Base de Datos en phpMyAdmin
  1. En tu navegador, ve a: http://localhost/phpmyadmin
  2. Haz clic en "Nueva" (New).
  3. Escribe el nombre de tu base de datos (ejemplo: db_proyecto).
  4. Haz clic en "Crear".
3. Configurar el archivo .env

Abre tu proyecto en VS Code, busca el archivo .env en la raíz y modifica estas líneas:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=db_proyecto  <-- Pon aquí el nombre que creaste
DB_USERNAME=root         <-- Usuario por defecto en XAMPP
DB_PASSWORD=             <-- Déjalo vacío (en XAMPP no tiene clave)
Importante: Después de cambiar el archivo .env, es recomendable limpiar la caché con:
php artisan config:clear
php artisan config:cache

Verificar Conexión y Crear Tablas

Una vez configurado el archivo .env, debemos comprobar si Laravel puede comunicarse con MySQL ejecutando las migraciones iniciales.

Ejecutar Migración

Abre la terminal (dentro de VS Code o en el CMD) y escribe:

php artisan migrate
¿Cómo saber si funcionó?

En la Terminal:

Preparing database...
Creating migration table...
Running migrations... DONE

En phpMyAdmin:

Refresca tu base de datos; deberías ver tablas como users, jobs, migrations, etc.

Errores Comunes:
  • "Unknown database": El nombre en DB_DATABASE no coincide con el de phpMyAdmin.
  • "Connection refused": MySQL en el Panel de XAMPP no está encendido (botón Start).
  • "Access denied": Revisa que el DB_USERNAME sea root y el password esté vacío.

Creación de Vistas (Blade)

Las vistas en Laravel deben terminar con: 
.blade.php
Ejemplo: 
resources/views/productos/index.blade.php

Modelo y $fillable


class Producto extends Model
{
    protected $fillable = [
        'nombre',
        'precio',
        'descripcion'
    ];
}
Permite insertar datos de forma segura usando create().

Rutas CRUD


Route::get('/productos', [ProductoController::class, 'index'])
    ->name('productos.index');

Route::get('/productos/create', [ProductoController::class, 'create'])
    ->name('productos.create');

Route::post('/productos', [ProductoController::class, 'store'])
    ->name('productos.store');

Uso de route() en Formularios


<form action="{{ route('productos.store') }}" method="POST">
    @csrf
</form>

->name() permite renombrar rutas y usarlas fácilmente.

foreach en Blade


@foreach ($productos as $producto)
    <p>{{ $producto->nombre }} - {{ $producto->precio }}</p>
@endforeach
